Parallelization

R by default doesn't make use of multiple cores. With the integration of parallelMap into mlr, it becomes easy to activate the parallel computing capabilities already supported by mlr.

library("mlr")
library("parallelMap")
parallelStartSocket(2)

## Starting parallelization in mode=socket with cpus=2.

task = makeClassifTask(data = iris, target = "Species")
lrn = makeLearner("classif.lda")
rdesc = makeResampleDesc("CV", iters = 4)
r = resample(lrn, task, rdesc)

## Exporting objects to slaves for mode socket: .mlr.slave.options
## Mapping in parallel: mode = socket; cpus = 2; elements = 4.
## [Resample] Result: mmce.test.mean=0.0265

parallelStop()

## Stopped parallelization. All cleaned up.

In Linux or Mac OS X, you may want to use parallelStartMulticore() instead.

For further ways of parallelizing your computations, consult the parallelMap tutorial and help.
